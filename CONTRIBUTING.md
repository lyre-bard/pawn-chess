# Contributing to Pawn Chess

Thank you for your interest in contributing to Pawn Chess! We welcome all kinds of contributions:

- 🐛 **Bug reports** - Found a bug? Report it!
- 💡 **Feature suggestions** - Have a cool idea? Let us know!
- 🔧 **Code improvements** - Help us improve the code quality
- 📝 **Documentation** - Improve our docs and guides
- 🎨 **UI/UX enhancements** - Make the game more beautiful or easier to use

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/pawn-chess.git
   cd pawn-chess
   ```
3. **Create a new branch** for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Making Changes

- Keep changes focused and atomic (one feature per PR)
- Test your changes thoroughly in multiple browsers
- Keep code clean and readable
- Add comments for complex logic

## Code Style

- Use meaningful variable names
- Keep functions small and focused
- Add comments for non-obvious logic
- Test on desktop and mobile

## Submitting Your Contribution

1. **Commit your changes** with clear messages:
   ```bash
   git commit -m "Add feature: description of what you added"
   ```

2. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

3. **Open a Pull Request** on the main repository with:
   - Clear title describing the change
   - Description of what you changed and why
   - Link to any related issues (if applicable)

## Reporting Bugs

When reporting bugs, please include:
- What you were doing when the bug occurred
- What you expected to happen
- What actually happened
- Browser and device information
- Steps to reproduce the issue

## Feature Suggestions

We love feature ideas! When suggesting a feature:
- Explain the use case
- Describe how it would improve the game
- Provide mockups or examples if possible

## Code Review

All submissions require code review. A maintainer will:
- Review your code for quality and correctness
- Test your changes
- Suggest improvements if needed
- Merge when approved

## Questions?

Feel free to open an issue for any questions or discussions!
